## Parte 4: Utilidade nos Sistemas de Informação


### 1. Utilidade da aplicação para Sistemas de Informação

- Que **dados** o sistema coleta?
- Como esses dados são **transformados em informação útil**?
- Quem **usa essas informações**?
- Que **decisões podem ser tomadas com elas**?

### 2. Tipo de Sistema de Informação

##### Sistemas Operacionais (TPS – Transaction Processing Systems)

Usados no dia a dia da operação.

Exemplos:

- sistema de vendas
- sistema de pedidos
- sistema de estoque
- sistema de pagamentos

Processam **grande volume de transações**.

---

##### Sistemas Gerenciais (MIS – Management Information Systems)

Geram **relatórios e análises para gestores**.

Exemplos:

- relatórios de vendas
- desempenho de produtos
- análise de estoque

Transformam dados operacionais em **informação gerencial**.

---

##### Sistemas de Apoio à Decisão (DSS)

Ajudam gestores a tomar decisões.

Exemplos:

- previsões  
- simulações
- análise de cenários

### 3. Relação com a Cadeia de Valor

A **cadeia de valor (Michael Porter)** mostra como a empresa cria valor.

Ela possui atividades como:

#### Atividades primárias

- logística
- operações
- marketing e vendas
- serviço ao cliente

#### Atividades de suporte

- infraestrutura
- tecnologia
- recursos humanos

O requisito pede para explicar **em quais partes da cadeia de valor o sistema ajuda**.

### 4. Uso Estratégico da Informação

#### Redução de custos

Automatizando processos.

Exemplo:

- menos trabalho manual
- menos erros

---

#### Diferenciação

Oferecendo algo que concorrentes não têm.

Exemplo:

- rastreamento do pedido  
- relatórios inteligentes
- integração com plataformas

---

#### Melhor tomada de decisão

Usando dados para orientar estratégia.

Exemplo:

- identificar produtos mais lucrativos
- entender horários de maior demanda

### 6. Requisitos Técnicos

- Manter o HTML e o CSS dentro de uma mesma pasta, em arquivos com extensões .html e .css, sem utilizar caracteres especiais ou acentuação nos nomes dos arquivos.
- Implementar um menu de navegação com links para as diferentes páginas do website, presente em todas as páginas para facilitar a navegação.
- Utilizar uma tabela para apresentar alguns dados pesquisados sobre games ou informações relevantes ao Happy Game.
- Inserir imagens, vídeos e outros elementos multimídia relevantes, utilizando as tags HTML apropriadas. A parte ilustrativa deve ser feita com arquivos locais para que o website funcione offline.
- Estilização com CSS:
    - Escrever o código CSS para formatar os elementos do website, aplicando estilos, cores, fontes e o layout definido no design.
    - Utilizar variáveis CSS para facilitar a manutenção do código e garantir a consistência visual.
    - Implementar media queries para ajustar o layout e os estilos de acordo com o tamanho da tela do dispositivo.
    - Criar um layout atraente com fotos, imagens e ícones (usando o bom senso), garantindo uma identidade visual única entre as páginas.
    - Formulário e Feedback:
        - Criar um formulário na Página Interna 3, utilizando a tag <form> e diferentes tipos de campos de entrada (<input>), para simular cadastro ou contato.
        - Utilizar atributos HTML para definir tipos de dados, nomes dos campos e validação básica dos dados inseridos.
        - Configurar o formulário para direcionar o usuário para a Página Interna 4 após o envio, mostrando uma mensagem de sucesso. Esta é uma simulação; não é necessário processar os dados.
